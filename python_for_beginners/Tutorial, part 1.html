Python

Python, by itself, is not a good scietific programming language. It is dreadfully slow and based mostly on for loops -- not much is vectorized. The main libraries and packages that make it good are NumPy, SciPy and matplotlib (as well as many others). The base library, NumPy, contains fast routines for common mathematic and scientific operations. SciPy is another set of more advanced routines that depend on NumPy, and matplotlib is for visualizing your data. It should also be mentioned that Pandas and scikit-learn are good for input/output and writing really advanced algorithms (academics: that means you).

To possibly speed up your (already optimized!) code, Numba is just-budding JIT and Cython is a long-existing C interface. When your code is already all-vectorized, Numba doesn't help much, but if you even have one for-loop, Numba helps. But be warned: it's very new, hence it doesn't work with all the code yet. If your code runs normally, but throws errors under Numba, don't use Numba.

This is just the surface of the packages available. To get the whole scientific stack, look at Anaconda or the Enthought Python distro for full Python distros you can modify, or, for an IDE, look at Spyder.

The basics

Python is not like Matlab. With Matlab, there are weird things such as that you can't have a script and function definitions in the same file. With Python, it's just as if you running everything at the prompt, meaning you can have declarations and use of those declarations in the same file.

Booleans

Besides True and False, the biggest thing here is None. If you have return None, that means that nothing is returned; it just ends the definition.

Importing functions

Let's say we have two files, called script.py and functions.py. script.py, as the name suggests, is just a script that uses functions in functions.py. To use these functions, we can write

  • from functions import *-- imports all the functions in the global namespace
  • import functions as f -- means we can do some_value = f.some_function_name(...)
  • from functions import some_function_name -- only import some_functions_name.

Docstrings

When you define a function, a doc-string shows up in other interfaces to describe the function. The begin and end with three " right below the declaration.

Your interpetter

Use the IPython interpetter. It gives you tab-completion, history-completion, doc-strings, magic shell commands, all that stuff. The biggest one I use is %run <filename>. That puts all of the variables in your namespace, meaning you can see stuff inside your script.

When I'm writing a function, I don't include it in a def ...: at first. Instead, write my function just as a script. This means that I can use IPython's %run to ensure that my function is running right. After I've written it, I wrap it in a function.
